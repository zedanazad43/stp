## Architecture

### Backend (server.js)

The Express server provides REST API endpoints for language management:

- **GET `/api/languages`**: Returns list of all supported language codes
- **GET `/api/locale/:lang`**: Returns translation JSON for a specific language

Example usage:
```bash
# Get available languages
curl http://localhost:8080/api/languages
# Returns: {"languages":["en","de","ar","zh","fr","es"]}

# Get English translations
curl http://localhost:8080/api/locale/en
# Returns: Complete en.json translation file
```

### Frontend (docs/index.html)

The HTML page includes:
1. **Language Selector**: Floating dropdown in the top-right corner
2. **Dynamic Translation Loading**: JavaScript fetches and applies translations on-the-fly
3. **LocalStorage Persistence**: Remembers user's language preference

The page automatically:
- Loads the user's preferred language from localStorage
- Falls back to English if a language fails to load
- Updates all text elements when language changes
- Preserves the selection across page reloads

## Adding a New Language

To add support for a new language:

1. **Create a new JSON file** in `locales/` directory:
   ```bash
   cp locales/en.json locales/xx.json
   ```

2. **Update the translations** in the new file with native language text

3. **Add the language code** to `server.js`:
   ```javascript
   const SUPPORTED_LANGUAGES = ["en", "de", "ar", "zh", "fr", "es", "xx"];
   ```

4. **Add the language option** to `docs/index.html`:
   ```html
   <option value="xx">üè≥Ô∏è Language Name</option>
   ```

5. **Update README.md** with the new language sections

6. **Test** the new language by selecting it from the dropdown

## Translation Guidelines

When translating content:

1. **Maintain Consistency**: Keep terminology consistent across all sections
2. **Cultural Adaptation**: Adapt idioms and expressions to local culture
3. **Technical Terms**: Some technical terms (like "Blockchain") may stay in English or be transliterated
4. **Character Encoding**: Ensure UTF-8 encoding for non-Latin scripts
5. **RTL Support**: For right-to-left languages (Arabic), the HTML already has proper `lang` attribute support

## Testing

To test the internationalization:

1. **Start the server**:
   ```bash
   npm install
   npm start
   ```

2. **Open the documentation page**:
   ```bash
   # In a separate terminal
   cd docs
   python3 -m http.server 8081
   ```

3. **Visit**: http://localhost:8081/index.html

4. **Test language switching**: Use the dropdown in the top-right corner

5. **Verify API endpoints**:
   ```bash
   curl http://localhost:8080/api/languages
   curl http://localhost:8080/api/locale/en
   ```
